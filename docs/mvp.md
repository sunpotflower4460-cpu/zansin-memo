# MVP

## MVPの目的
Kizashi Notes の核となる体験（書く・残る・再会する・育てる）を、最小機能で検証すること。

## 必須機能
1. 種を書く
2. 種一覧を見る
3. 種の詳細を見る
4. 種に状態をつける（seed / sprout / tree / archived）
5. 種に気分・カテゴリ・重要度をつける
6. 今日の種をランダムまたはスコア順に1つ浮上させる
7. 種を「問い」「タスク」「記事案」「プロジェクト案」に変換するUI土台
8. ローカル保存または簡易DB保存を想定した設計

## 初期でやらないこと
- 複雑なAI連携
- ログイン必須化
- 課金
- SNS機能
- 重い3D表現
- 完全自動分類
- 複雑なグラフDB
- 過剰なアニメーション

## 成功条件
- 一言入力でも継続的に種を記録できる
- 数日後に過去の種と再会できる
- 種を次行動へ変換する導線がある
- UIが静かで低圧なまま成立している

## MVP完了チェックリスト
- [ ] Home / Write / Seeds / Seed Detail / Garden の最小画面がある
- [ ] Seed データモデルが一貫している
- [ ] ローカル保存で CRUD が成立する
- [ ] 状態・気分・カテゴリ・重要度が保存できる
- [ ] 再浮上ロジック（ランダム or スコア）が動作する
- [ ] 変換アクションUIの土台がある
- [ ] コンセプトから逸脱していない
