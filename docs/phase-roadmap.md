# Phase Roadmap

## Phase 0: Docs Foundation
### 目的
- コンセプトとMVPを固定し、実装前提を揃える

### やること
- README / docs / AGENTS / copilot-instructions を整備
- 用語（seed, sprout, tree, resurfacing）を統一

### やらないこと
- 実装作業
- 依存追加

### 完了条件
- 主要設計ドキュメントが揃っている
- 後続エージェントが迷わず実装に入れる

### 次Phaseへの受け渡し
- 画面要件・MVP要件・非目標が明文化されている

---

## Phase 1: App Shell
### 目的
- 最小UIの土台を作る

### やること
- Home / Write / Seeds / Detail の画面枠を作成
- 仮データ表示で画面遷移を確認

### やらないこと
- 永続化
- 複雑ロジック

### 完了条件
- 主要画面へ移動できる
- 情報設計が docs と一致している

### 次Phaseへの受け渡し
- 画面構造とコンポーネント分割方針

---

## Phase 2: Seed Data Model
### 目的
- Seed中心のデータ基盤を確立する

### やること
- Seed型を定義
- 状態管理を実装
- ローカル保存を導入

### やらないこと
- クラウド同期
- AI連携

### 完了条件
- CRUD が成立し、再読込で保持される

### 次Phaseへの受け渡し
- 安定したデータアクセス層

---

## Phase 3: Write Experience
### 目的
- 書くハードルを最小化する

### やること
- 一言入力を中心にした作成UI
- 気分・重要度・状態の入力導線
- 安心感のある文言設計

### やらないこと
- 入力時の強制分類
- 高機能エディタ化

### 完了条件
- 数秒で種を保存できる
- 入力体験が低圧である

### 次Phaseへの受け渡し
- 作成データが一覧・詳細で再利用可能

---

## Phase 4: Seed List & Detail
### 目的
- 見返しと編集の中核体験を作る

### やること
- 一覧表示
- 検索 / フィルター / 並び替え
- 詳細表示と更新

### やらないこと
- 高度な可視化
- 自動推論の多用

### 完了条件
- 必要な種に素早く再アクセスできる

### 次Phaseへの受け渡し
- 再浮上対象を選べるデータ土台

---

## Phase 5: Gentle Resurfacing
### 目的
- 過去の種との再会を設計する

### やること
- 今日の種表示
- resurfacingScore導入
- 古い種のやさしい提示

### やらないこと
- 強い通知圧
- ブラックボックス判定

### 完了条件
- 継続利用時に再発見体験が生まれる

### 次Phaseへの受け渡し
- 変換アクションに繋がる再浮上導線

---

## Phase 6: Transform Actions
### 目的
- 種を次のアウトプットへ接続する

### やること
- 問いにする
- タスクにする
- 記事案にする
- プロジェクト案にする
- 初期はテンプレ変換で実装

### やらないこと
- 生成AI前提の重機能
- 自動確定

### 完了条件
- 各変換がUIから選択・保存できる

### 次Phaseへの受け渡し
- Garden表示で使える関係情報

---

## Phase 7: Garden View
### 目的
- 種・芽・木の関係を視覚的に把握できるようにする

### やること
- カード/簡易マップ表示
- 状態ごとの視認性調整

### やらないこと
- 重い3D表現
- 複雑なグラフエンジン

### 完了条件
- 視覚的にテーマのまとまりが把握できる

### 次Phaseへの受け渡し
- AI補助の導入ポイント整理

---

## Phase 8: AI Assist Preparation
### 目的
- AI補助を安全に追加できる準備を整える

### やること
- AI連携インターフェース定義
- ローカルフォールバック設計
- セキュリティ方針確認

### やらないこと
- APIキー必須化
- 課金・認証の導入

### 完了条件
- AIなしでも全機能が成立
- 将来のAI差し込み点が明確

### 次Phaseへの受け渡し
- 本格AIフェーズの技術検証計画
