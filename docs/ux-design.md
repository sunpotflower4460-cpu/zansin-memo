# UX Design

## 画面一覧
1. **Home**
   - 今日浮上した種
   - 最近書いた種
   - いま育っているテーマ
   - 新規メモ導線
2. **Write**
   - 一言入力ができる最小フォーム
   - 気分・重要度・カテゴリの簡易入力
   - 「まだ種でよい」安心感の提示
3. **Seeds**
   - 種一覧
   - 状態別フィルター
   - 検索
   - 並び替え
4. **Seed Detail**
   - 本文・状態・メタ情報
   - 関連種
   - 次の問い
   - 変換候補
5. **Garden**
   - 種・芽・木の軽量ビジュアル
   - MVPはカード型中心

## ユーザーフロー
1. Home から新規作成へ
2. Write で断片を保存
3. Seeds / Detail で見返し・編集
4. Home で再浮上した種と再会
5. Transform Action で次の行動へ接続

## 書く体験
- 入力開始までの手数を最小化
- タイトル必須にしない
- 後から編集できる前提で心理負担を下げる
- 完成文章を要求しない

## 見返す体験
- 状態・気分・カテゴリで軽く絞り込める
- 直近と過去をバランスよく提示する
- 重要な種に再アクセスしやすい

## 種が浮上する体験
- 日次で1件以上をやさしく提示
- 通知的な圧ではなく、提案として表示
- 理由の説明（例: 「しばらく見返していない」）を添える余地を残す

## AIが入る場合の距離感
- AIは提案のみ行い、確定操作はユーザーが選ぶ
- UI上でAIを主役化しない
- オフライン相当でも成立する設計を優先

## UIトーン
- 静か
- やさしい
- 余白がある
- 書くことを急かさない
- 創造性を邪魔しない

## アクセシビリティ配慮
- 十分なコントラスト比
- キーボード操作対応
- フォーカス可視化
- スクリーンリーダー向けラベル
- 色だけに依存しない状態表現
